PaperTrail Scripting

Papertrail exposes a number of services and objects that are available to use in scripts e.g. runScript rule etc..

Document

The DocumentModel is the core object that will be used for scripting it will be available in most scripts in the doc variable

It is a Java Class of type com.egis.DocumentModel that also support the groovy dynamic functions e.g.

e.g. the following are all equivalent.

doc.metadata().get('index1')
doc.index1
doc["index1"]

Any document property can used.

Prior to 8.3.3 doc.save() needed to be called in order to ensure that the indexes and document grid were correctly updated, however from 8.3.3 doc.save() is optional and only needs to be used to force a flush to the database. Any unsaved documents will be saved on database commit

See DocumentModel

Session

The session corresponds to the request and is bound to a specific user which is accessible via session.user A documents session is available via doc.session

And from anywhere via:

com.egis.kernel.Kernel.get(com.egis.Session.class)

See Session

Querying Documents

session.getDocument(1); // get document by docId
session.getDocument('1'); // get document by docId
session.getDocument('System/templates/test.txt'); // get document by path
session.getDocument('Invoices/invoiceNo=123'); // get document by query
session.getDocument("SELECT * FROM 'Invoices' WHERE invoiceNo = 123"); // get document by PQL

Session.query

Session.query has the following syntax: session.query(query, clazz) where clazz is one of:

  • DocumentModel.class - the default and equivalent to session.query(query)
  • Map.class
  • String.class
  • Long.class
  • com.egis.utils.js.JsArray.class
  • Result.class

iterate through a list of documents return from a search

session.query("SELECT * FROM 'Invoices'").each {doc -> 
     println doc.filename
}

quering for data

session.query("SELECT index1,index2 FROM 'Invoices'", Map.class).each { map -> 
    assert map.size() == 2;
    assert map.index1 != null && map.index2 !- null
}

Importing Documents

InputStream in = new ByteArrayInputStream("test".bytes);
session.createDocument('System/templates', 'test.txt', in)

or 2 steps

doc =  session.createDocument('System/templates', 'test.txt')
doc.title = 'Default TXT Template'
doc.content().write(in);

// which would result in
in === doc.content().read()

creating an item

doc = session.createItem('System/templates', 'Item)

Links

Linked Documents on a DocumentModel are accessible from doc.links()

checking if a doc has links

doc.links().hasAny()

//returns a Boolean

checking if a doc is linked to another

doc.links().contains(session.getDocument(1))
doc.links().contains('System/templates/test.txt')

//returns a Boolean

adding a link

doc.links().add(session.getDocument(1))
doc.links().add('System/templates/test.txt')

removing a link

doc.links().remove(session.getDocument(1))
doc.links().remove('System/templates/test.txt')

getting all links with link data

doc.links().getAllData()

//returns a list of <LinkDocument>

iterating over all links as LinkDocument

doc.links().getAllData().each {
    println it.filename //linked document filename
    println it.direction //link direction
}

getting all links as DocumentModel

doc.links().getAll()

//returns a list of <DocumentModel>

iterating over all links as DocumentModel

doc.links().getAll().each { link ->
    println link.createdBy
}

You can import any class in  any jar file available in the System/jars (recommended for production) or on the classpath that PaperTrail was started with.

See JavaDocs for all classes in the public API.
